# 전시회장 지도 인터랙티브 웹 변환 프로젝트 - 핵심 기술 및 개념 정리

이 문서는 단순 픽셀 이미지(`map.png`)에서 부스 데이터를 추출하고, 이를 웹에서 편집 가능한 인터랙티브 지도로 변환하는 과정에서 사용된 주요 프로그래밍 및 수학적/시각적 개념들을 정리한 것입니다.

## 1. 이미지 처리 및 컴퓨터 비전 (Python, OpenCV)

사진에서 기계가 의미 있는 데이터(부스 위치, 크기)를 스스로 찾아내기 위해 사용된 기술들입니다.

- **색상 공간 변환 (Color Space Conversion)**:
  - 인간이 보는 RGB(또는 BGR) 포맷 대신, 빛의 밝기나 그림자에 덜 민감한 **HSV(Hue, Saturation, Value; 색상, 채도, 명도)** 공간으로 이미지를 변환했습니다.
  - 이를 통해 특정 색상(채도가 높은 부스 구역)이나 회색 외곽선(채도가 낮고 명도가 중간인 영역), 흰색 배경 등을 정확하게 필터링할 수 있었습니다.
  
- **임계값 처리 및 마스킹 (Thresholding & Masking)**:
  - 특정 조건(예: 채도가 15 이상인 픽셀만 1(흰색), 나머지는 0(검은색)으로 설정)을 부여하여 우리가 원하는 부스 영역만 흑백 지도로 뽑아내는 기법입니다.

- **모폴로지 연산 (Morphological Operations - Closing)**:
  - 이미지 상의 부스들은 얇은 흰색 선(그리드)으로 쪼개져 있었습니다. 이를 하나의 거대한 부스 덩어리로 인식하게 하기 위해, 픽셀을 확장(Dilation)했다가 다시 깎아내는(Erosion) '닫기(Close)' 연산을 수행했습니다.

- **윤곽선 검출 (Contour Detection)**:
  - `cv2.findContours` 알고리즘을 사용하여 마스킹된 덩어리들의 외곽선을 따라가며 X, Y 좌표와 가로(Width), 세로(Height) 픽셀 크기를 수학적인 상자(Bounding Box) 데이터로 추출했습니다.

- **그리드 투영 및 분석 (Grid Projection)**:
  - 추출한 부스 크기들이 불규칙하므로 핵심 '최소 1칸(단위)'의 픽셀 크기를 찾아야 했습니다.
  - 이미지의 픽셀값들을 가로축과 세로축으로 더해 평균을 내는 투영(Projection) 프로파일 방식을 사용하여, 얇은 흰색 선들이 일정한 픽셀(45px) 간격으로 나타난다는 패턴을 수학적으로 찾아내어 1 unit = 45px 임을 알아냈습니다.
  
- **색상 거리 계산 및 클러스터링 (Color Distance & Clustering)**:
  - 추출된 1 unit 단위의 칸들을 순회하며 서로 인접해 있고, **색상 차이(유클리디안 거리)** 가 기준치 이하로 비슷한 칸들은 똑같은 회사의 부스로 간주하여 하나의 큰 블록(Block)으로 병합(Merge)하는 알고리즘을 작성했습니다.

## 2. 인터랙티브 웹 프론트엔드 (HTML/CSS/JavaScript)

추출된 정적 데이터(`JSON`)를 마우스로 조작 가능하고 정보 저장이 가능한 생동감 있는 지도 인터페이스로 구현한 기술들입니다.

- **절대 위치 및 그리드 매핑 (Absolute Positioning)**:
  - 파이썬이 뽑아준 X=3, Y=4 와 같은 단위 좌표에 `CSS 단위 픽셀(45px)`을 곱해 `left: 135px`, `top: 180px` 처럼 CSS 절대 좌표로 환산하여 화면에 부스를 렌더링했습니다.

- **마우스 이벤트와 드래그 앤 드롭 (Mouse Events & Drag-and-Drop)**:
  - 자바스크립트의 `mousedown`, `mousemove`, `mouseup` 이벤트를 조합하여 마우스를 누른 채 움직일 때 부스의 `left`, `top` 값이 실시간으로 변경되게 하였습니다. (드래그 알고리즘)

- **그리드 스냅 (Grid Snapping)**:
  - 부스를 마음대로 움직여도 마우스를 뗄 때는 항상 45px 격자 안에 딱 맞춰서 들어가도록(Snapping) `Math.round(위치 / 45) * 45` 라는 수학 연산을 적용하여 깔끔한 레이아웃이 유지되게 했습니다. 동일한 원리로 우측 하단 조절바(Resize Handle)를 통한 크기 조절도 구현했습니다.
  
- **로컬 스토리지 (Local Storage API)**:
  - 보통 데이터를 저장하려면 복잡한 백엔드 서버와 데이터베이스가 필요하지만, 이번 프로젝트는 브라우저 내부에 데이터를 영구 저장할 수 있는 `Window.localStorage` API를 활용했습니다.
  - 덕분에 새로고침을 하거나 브라우저를 껐다 켜도 수정한 레이아웃이나 타이핑해둔 회사 정보가 그대로 남아있게 됩니다.

- **자바스크립트 기반 JSON 파일 다운로드 (Data URI & Blob)**:
  - 사용자가 "Download JSON"을 클릭했을 때 브라우저 메모리에 있는 데이터를 마치 실제 파일인 것처럼 변환(`data:text/json;charset=utf-8,...`)하여 다운로드를 강제 실행(가상 앵커 태그 원리)시킴으로써, 서버 통신 없이도 백업 파일을 만들 수 있도록 구현했습니다.
